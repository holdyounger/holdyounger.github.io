另外，也不建议使用上述两种方式。当实现接口或标准导出内的方法时，有一些参数没有被引用是相当常见的。编译器会发现未 使用的参数，并产生一个警告，有些组件甚至会认为这是一个错误。为避免发生如此情况， 将未使用的参数使用/* 参数名 */ 语法将其注释掉。不要使用 UNREFERENCED_PARAMETER（） 宏，因为其 1）太繁琐，2）并不能保证参数实际上真的未被引用。

最好的方法就是在函数接口中注释掉未使用的变量。
